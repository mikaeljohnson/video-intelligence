## 📋 The Prompt (Copy-Paste Ready)

```
You are a senior cloud solutions architect. Build a production-ready,
serverless video and audio intelligence pipeline on AWS using
Infrastructure as Code (CloudFormation or Terraform).

The pipeline must handle the following end-to-end flow:
1. User uploads a video file (MP4, MOV, MKV, AVI, WebM) to an S3
   "raw" bucket under the prefix raw-videos/
2. An S3 PutObject event triggers a Lambda function via SNS fan-out
3. That Lambda calls AWS MediaConvert to create adaptive bitrate
   H.264/MP4 outputs at 1080p (5 Mbps), 720p (2.5 Mbps), and 480p
   (1 Mbps)
4. MediaConvert's completion event (via EventBridge) triggers a
   second Lambda
5. The second Lambda runs AWS Rekognition analysis: label detection,
   content moderation, and (optionally) celebrity recognition
6. The second Lambda also runs Amazon Transcribe to generate a WebVTT
   (.vtt) subtitle file from the video's audio track
7. All outputs — transcoded videos, WebVTT subtitles, and a structured
   JSON moderation report — are written to a "processed" S3 bucket
8. An SES email notification is sent with direct S3 links to all
   artifacts and a moderation flag table
9. If SES fails, an SNS notification fires as fallback
[... full prompt continues in PROMPT.md ...]
```

## 🏗️ What the Prompt Generates

When a developer runs this prompt, they get:

| Component | Details |
|---|---|
| **S3 Buckets** | Raw uploads bucket + processed outputs bucket (SSE-S3, versioning, lifecycle) |
| **Lambda Functions** | `VideoPipeline-Trigger` + `VideoPipeline-Process` (Python 3.11, fully implemented) |
| **IAM Roles** | MediaConvert service role + Lambda execution role (least-privilege) |
| **EventBridge Rule** | MediaConvert completion → Lambda routing with input transformer |
| **CloudWatch Alarms** | Pipeline error, MediaConvert lag, Rekognition lag, SES bounce rate |
| **SES/SNS Notifications** | HTML email with S3 links + moderation flag table, SNS fallback |

**Output per video processed:**
- `transcoded/{name}_hd.mp4` — 1920×1080 @ 5 Mbps
- `transcoded/{name}_sd.mp4` — 1280×720 @ 2.5 Mbps
- `transcoded/{name}_mobile.mp4` — 854×480 @ 1 Mbps
- `subtitles/{name}.vtt` — WebVTT captions
- `reports/{name}-report.json` — Timestamped labels, moderation flags, metrics
- 📧 Email notification with direct links to all artifacts

## 🚀 Quick Start

### 1. Copy the prompt

**[→ Open PROMPT.md](PROMPT.md)** and copy the full text.

### 2. Paste it into an AI assistant

```bash
# Deploy infrastructure with the prompt's output
aws cloudformation deploy \
  --stack-name media-intelligence \
  --template-file cfn/pipeline.yaml \
  --capabilities CAPABILITY_IAM \
  --parameter-overrides \
    NotificationEmail=you@example.com \
    EnableCelebrityRecognition=false \
    ModerationThreshold=75
```

### 3. Upload a test video

```bash
aws s3 cp test-video.mp4 \
  s3://media-intelligence-raw-videos-<ACCOUNT>-<REGION>/raw-videos/test-video.mp4
```

### 4. Watch it process

Check your email — you'll receive a completion notification with links to:
- 3 transcoded H.264 outputs
- WebVTT subtitle file
- JSON moderation report with timestamped flags
